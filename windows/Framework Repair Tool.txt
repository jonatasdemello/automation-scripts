Microsoft .NET Framework Repair Tool.

NetFxRepairTool.exe /option <parameter>

Options:
	/? | /help
		Help information.

	/q[uiet]
		Quiet mode - No user interaction.
		Options '/q' and '/p' are mutually exclusive.

	/p[assive]
		Unattended mode - Progress bar only.
		Options '/q' and '/p' are mutually exclusive.

	/a[ddsource]
		Offline repair mode - Setup redistributables required for
		repair are copied from a network share or folder path specified by the '/addsource' flag.

		Example:
			NetFxRepairTool.exe /a "\\myserver\redists"

		.NET Framework 2.0 SP2 and .NET Framework 3.0 SP2 can be repaired using .NET Framework 3.5 SP1 redistributable package.
		Repairing .NET Framework 3.0 SP2 and .NET Framework 3.5 SP1 are cumulative (i.e. repair 2.0 SP2 and 3.0 SP2).

	/r[epair]
		Scoped repair:
		Limits repairing .NET Framework 4.0, .NET Framework 3.5 or nothing (targeted fixes are still applied).

		Default (not specifying this flag) will repair all supported .NET Frameworks installed.

		This option is only applicable in quiet and passive modes.
		When this option is specified as 'None', the '/addsource' option should not be specified.
		Example:

		- To repair .NET Framework 4.8 only, use the following:
			NetFxRepairTool.exe /q /r .Net48

		- To repair .NET Framework 4.7.2 only, use the following:
			NetFxRepairTool.exe /q /r .Net472

		- To repair .NET Framework 4.7.1 only, use the following:
			NetFxRepairTool.exe /q /r .Net471

		- To repair .NET Framework 4.7 only, use the following:
			NetFxRepairTool.exe /q /r .Net47

		- To repair .NET Framework 4.6.2 only, use the following:
			NetFxRepairTool.exe /q /r .Net462

		- To repair .NET Framework 4.6.1 only, use the following:
			NetFxRepairTool.exe /q /r .Net461

		- To repair .NET Framework 4.6 only, use the following:
			NetFxRepairTool.exe /q /r .Net46

		- To repair .NET Framework 4.5.2 only, use the following:
			NetFxRepairTool.exe /q /r .Net452

		- To repair .NET Framework 4.5.1 only, use the following:
			NetFxRepairTool.exe /q /r .Net451

		- To repair .NET Framework 4.5 only, use the following:
			NetFxRepairTool.exe /q /r .Net45

		- To repair .NET Framework 4.0 only, use the following:
			NetFxRepairTool.exe /q /r .Net4

		- To repair .NET Framework 3.5 SP1 (including .NET 3.0 SP2 and.NET 2.0 SP2) only, use the following:
			NetFxRepairTool.exe /q /r .Net35
		  This command can also be used to repair .NET 3.0 SP2 and / or .NET 2.0 SP2, when .NET 3.5 SP1 is not installed.

		- To skip repair operation, use the following:
			NetFxRepairTool.exe /q /r None

		- To repair more than one .NET Frameworks, the values should be separated by ';', for example:
			NetFxRepairTool.exe /q /r .Net35;.Net45

		  However, since .NET Framework 4.* setups are in-place
		  major upgrades (e.g. 4.5.1 major upgrades both 4.5 and 4.0),
		  only one of the specified 4.* frameworks can be repaired.

	Repair operation can be optionally scoped to target a set of specified product baselines.
	If this option is not specified, the tool will try to repair all supported .NET Framework versions installed.
	The "/repair" option can also be used together with "None" value to just apply fixes and skip running repair for any product baseline.
	When this option is used together with "None" value, then the "/addsource" option should not be specified.
	This option is applicable only in Quiet and Passive modes.

	NetFxRepairTool.exe /r <version1>; <version2>
	Supported version values: .net35, .net4, .net45, .net451, .net452, .net46, .net461, .net462, .net471, .net472, .net48, none
	Examples:
	NetFxRepairTool.exe /q /r .net4
	NetFxRepairTool.exe /q /r .net35;.net45


	/l[ogs] <folder path>
		Sets the desired folder path for the logs file cabinet
		generated by the tool. Default (not specifying this option)
		location of the cabinet file is current user's desktop folder.
		Example:
			NetFxRepairTool.exe /l "c:\MyLogsFolder"

	/c[ollectonly]
		Data collection only mode - Collects the logs and other
		information relevant to .NET Framework setup installations
		useful for troubleshooting. Repair and fix operations
		will not be performed. This option is applicable only in
		quiet and passive modes.

	/n[oceipconsent]
		Opt out of sending feedback information to Microsoft
		Corporation. Default (not specifying the flag) assumes
		that the user has opted in to send usage data to Microsoft
		Corporation. This option is applicable only in quiet
		and passive modes.
